//| 层级         | 校验内容   | 举例                    |
  | ---------- | ------ | --------------------- |
  | Controller | 参数格式校验 | 不能为空、长度、格式、枚举值        |
  | Service    | 业务规则校验 | 是否重复、是否存在、是否启用、是否允许操作 |
如果你只在 Controller 校验，那么其他地方绕过 Controller 直接调用 Service，就可能把错误数据写进数据库
//
Controller 层：
    校验“前端传来的格式对不对”
    比如不能为空、长度、格式、枚举值

Service 层：
    校验“业务上允不允许”
    比如是否存在、是否重复、是否启用、是否允许删除、是否允许修改

Mapper 层：
    不做业务校验，只负责数据库操作
