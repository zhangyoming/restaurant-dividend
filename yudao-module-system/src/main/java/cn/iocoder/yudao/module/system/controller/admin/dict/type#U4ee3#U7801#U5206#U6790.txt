\\前端请求接口
      ↓
  @GetMapping / @PostMapping 匹配接口地址

\\| 注解           | 来源                  | 默认注入方式    |
  | ------------ | ------------------- | --------- |
  | `@Resource`  | Java / Jakarta 标准注解 | 先按名称，再按类型 |
  | `@Autowired` | Spring 提供的注解        | 主要按类型     |
Spring 会先看变量名，然后去容器里找，如果找到了，就注入，如果按名字找不到，再尝试按类型
@Autowired Spring 主要会按类型
\\如果容器里只有一个实现类 二者没区别
假设一个接口有两个实现类 Spring 会懵 按类型找到了多个
所以在多个实现类时，@Resource 有时更直观
@Autowired 怎么解决多个实现类问题？
@Autowired
@Qualifier("wechatPayServiceImpl")
private PayService payService;
\\你现在看到的是字段注入：

  @Resource
  private DictTypeService dictTypeService;

  这种写法简单，很多项目都这么写。

  还有一种更推荐的现代写法是构造器注入：

  private final DictTypeService dictTypeService;

  public DictTypeController(DictTypeService dictTypeService) {
      this.dictTypeService = dictTypeService;
  }

  如果配合 Lombok，可以写：

  @RequiredArgsConstructor
  @RestController
  @RequestMapping("/system/dict-type")
  public class DictTypeController {

      private final DictTypeService dictTypeService;

  }
